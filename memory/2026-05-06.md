# 2026-05-06 — Mad Krapow Session Log

## Session Focus
Fixed Lalamove checkout delivery fee correctness for web and Flutter mobile.

## Key Finding
Flutter logs showed the quote request used fallback pickup coordinates `3.1390,101.6869` and default store address instead of the configured Shah Alam store. Web checkout had the same class of risk because client components imported server-only env values, which can fall back to defaults in client bundles.

## Decision
Lalamove pickup coordinates/address are now server-truth. Web and mobile quote requests should omit `pickup`; quote API routes resolve pickup from server env (`STORE_LATITUDE`, `STORE_LONGITUDE`, `STORE_ADDRESS`).

## Implementation Summary
- Added shared Lalamove quote helpers for server pickup resolution, coordinate formatting, price normalization, and cents conversion.
- Updated `/api/delivery/quote` and `/api/shipping/lalamove/quote` to use server pickup and normalized quote responses.
- Removed customer-facing Lalamove diagnostic leakage from quote errors.
- Updated web checkout and address input to stop sending client pickup coordinates.
- Updated Flutter checkout quote model/provider to stop sending pickup and to preserve quote expiry/service/price breakdown.
- Checkout creation now re-derives delivery fee from returned quote `priceBreakdown.total` when present.
